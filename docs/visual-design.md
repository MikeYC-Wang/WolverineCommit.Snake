# 視覺設計規格 (Visual Design Spec)

> 狀態：討論中，會隨對話持續更新。標記 `[TBD]` 的項目尚未拍板。

## 0. 已確認的關鍵決策

- 主題：**單一深色主題（dark theme only）**，v1 不做 light/dark 雙版本。
- 輸出格式：**只做 SVG 動畫**，不產出 GIF。
- 色彩方案：**方案 A（深藍 + 琥珀金）**，詳見第 3 節。

## 1. 整體風格定位

**方向：Wolverine 品牌風**

- 呈現「精通事件驅動（Event-Driven）後端工程師」的技術宣告感，視覺語彙貼近 Wolverine（.NET 事件驅動框架）官方識別。
- 背景仍維持 GitHub Contribution Graph 原本的綠色格子配色（v1 僅深色模式，見第 4 節），確保任何人一眼就能認出「這是我的 GitHub 貢獻牆」。
- 蛇本身（頭部、身體、Event 泡泡）套用 Wolverine 品牌配色，作為疊加在原始格子矩陣上的「事件驅動視覺層」。

## 2. 蛇的組成元件

### 2.1 蛇頭 = Command

- 造型：小方塊 / 圓角方塊，模擬一則「Command 訊息」。
- 語意：蛇頭移動到哪一格，代表一個 Command 被派送到該 Handler。
- 顏色：深藍（品牌底色）搭配亮橘/黃邊框或高光，作為視覺焦點。
- **方向感圖示（已確認：要）**：蛇頭加入極簡箭頭圖示，指向目前移動方向，強化「Command 被派送」的方向性語意。箭頭隨蛇頭轉彎即時旋轉。

### 2.2 蛇身 = 訊息匯流排 (Message Bus)

- 造型：節點以深藍色方塊呈現，節點間用細虛線／管線樣式連接，模擬事件在 bus 上傳遞的軌跡。
- 目的：讓「蛇」不只是遊戲畫面，而是一條可讀的事件傳遞管線。
- **長度與重置行為（已確認）**：蛇身維持固定節數（草案 10 節），吃格時頭進尾縮；跑完一輪所有 contribution 格子後重置回起始點、格子恢復原狀再開始下一輪。技術細節與理由見 `tech-stack.md` 第 3.1 節。

### 2.3 Event 泡泡（吃格動畫）

**已確認方向：氣泡 + 連線飛向蛇尾**

- 當蛇頭吃掉一格（= Handler 處理完畢）時：
  1. 該格位置冒出一顆小型氣泡（圓形，品牌強調色：橘/金）。
  2. 氣泡與蛇尾之間出現一條動態連線（虛線或漸層拖尾），模擬「事件被發布並傳遞到佇列尾端」。
  3. 動畫結束後氣泡與連線淡出消失。
- 因應 GitHub contribution 格子實際渲染尺寸極小（每格約 10–15px），氣泡內**不放文字**，改用極簡符號（例如一個小圓點或極簡閃電符號）作為「Event」的視覺代稱，避免在小尺寸下文字不可讀。
- **泡泡大小對應 contribution 強度（已確認：要）**：格子原本的 contribution 等級（GitHub 的 1~4 級，對應 `#0e4429` / `#006d32` / `#26a641` / `#39d353`）決定泡泡半徑與亮度——等級越高（顏色越亮），泡泡越大、越亮，象徵「這個 Handler 處理的事件量/重要程度越高」。
- **走訪範圍（已確認，見 `tech-stack.md` 第 3 節）**：蛇只在有 contribution 的格子間移動，不途經空白格。因此兩個連續吃格動作之間，蛇頭的移動距離可能跨越多格空白（非相鄰格）。渲染時蛇頭/身體的位移需用平滑的移動動畫（tween）銜接起點與終點，而非畫成沿格線逐格轉彎，避免看起來像瞬間移動。

## 3. 色彩計畫（方案 A：深藍 + 琥珀金，已確認）

| 元件 | 用途 | 色碼 | 備註 |
|---|---|---|---|
| 背景格子 | GitHub Dark Contribution 配色 | `#0e4429` / `#006d32` / `#26a641` / `#39d353` | 沿用 GitHub 深色模式綠階，不更動 |
| 蛇頭 (Command) | 焦點色 | fill `#1B2A4A`　border `#F5A623` | 深藍底、琥珀金邊框 |
| 蛇身 (Bus) | 次要色 | fill `#24345C`　connector `#F5A623` @ 40% opacity | 節點間虛線連接，模擬訊息匯流排 |
| Event 泡泡 | 強調色 | fill `#FFB000` | 吃格瞬間出現的金色氣泡 |
| 連線（泡泡→蛇尾） | 動態強調 | `#FFB000` @ 35% opacity，漸層拖尾 | 氣泡飛向蛇尾時的動態連線 |

風格關鍵字：deep navy / amber accent / 專業事件匯流排感。

## 4. 主題模式

- **v1 只做單一深色主題**，不提供 light/dark 雙版本，也不需要 `prefers-color-scheme` 切換邏輯。
- `[TBD]` 未來若要擴充 light 版本，需另外設計淺色背景下的蛇身/泡泡對比色（目前色碼是針對深色背景設計，直接套用在淺色背景可能對比不足）。

## 5. 動畫節奏（已確認）

- 每一步移動（蛇頭前往下一個 contribution 格，可能跨越多格空白，見第 2.3 節）：約 **200ms**，速度適中，肉眼可清楚辨識移動且不拖沓。
- Event 泡泡動畫序列：總長約 **600ms**
  1. 淡入 100ms
  2. 停留 200ms
  3. 連線飛向蛇尾 300ms
  4. 淡出（與步驟 3 尾端重疊）200ms
- 整輪跑完所有 contribution 格子：目標 **25–35 秒**。
- 跑完一輪後，蛇重置回起始點、格子恢復原狀（見第 2.2 節），停頓 **1–2 秒** 再開始下一輪，避免視覺上像「卡住」。
